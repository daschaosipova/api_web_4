# Космический Телеграм

Проект автоматизирует сбор «космических» фотографий и их публикацию в Telegram-канале. Скрипты умеют работать с несколькими источниками данных:
*   **SpaceX**: фотографии последних запусков ракет.
*   **NASA APOD**: ежедневные астрономические снимки дня.
*   **NASA EPIC**: уникальные кадры Земли, сделанные из глубокого космоса.

Проект позволяет как просто наполнять локальную папку крутыми обоями, так и вести полноценный тематический канал в автоматическом режиме.

## Как установить

### 1. Получение ключей
Для работы программы вам потребуются:
*   **NASA API Key**: Зарегистрируйтесь на [api.nasa.gov](https://nasa.gov) и получите токен. Обычно он выглядит как длинная строка символов (например, `u7H...F3q`).
*   **Telegram Bot Token**: Создайте бота через [@BotFather](https://t.me). В ответ вы получите токен вида `123456789:ABCdefGHI...`.
*   **ID канала**: Создайте канал в Telegram и добавьте туда бота как администратора. ID публичного канала выглядит как `@название_канала`.

### 2. Настройка окружения
Создайте в корневой папке проекта файл `.env`. В нем нужно сохранить ваши секретные ключи в таком формате:
```env
NASA_TOKEN=ваш_ключ_nasa
TG_SPACE_TOKEN=токен_вашего_бота
TG_CHANNEL=@имя_вашего_канала
PUBLISH_FREQUENCY=14400
EMAIL=рабочий адрес электронной почты для проверки от ботов
IMAGES_COUNT_NASA=50
```
*`PUBLISH_FREQUENCY` — это частота публикаций в секундах (по умолчанию 4 часа).*
*`IMAGES_COUNT_NASA` — это количество фото, которое вы хотите скачать в скрипте `fetch_apod_nasa_images.py`*

### 3. Установка зависимостей
Python3 должен быть уже установлен. Затем используйте pip (или pip3, если есть конфликт с Python2) для установки зависимостей:

```bash
pip install -r requirements.txt
```
## Запуск и возможности argparse

Скрипты поддерживают аргументы командной строки, что позволяет гибко управлять загрузкой и публикацией.

### Загрузка фото
Все изображения сохраняются в директорию `images/`.

*   **Снимки SpaceX**:
    ```bash
    python fetch_spacex_images.py --launch_id 5eb87d42ffd86e000604b384
    ```
    *Без аргумента `--launch_id` скрипт скачает фото последнего запуска.*

*   **NASA APOD (Astro Picture of the Day)**:
    ```bash
    python fetch_apod_nasa_images.py --images_count 10
    ```
    *Аргумент `--images_count` указывает, сколько случайных фотографий скачать.*

*   **NASA EPIC (Earth Polychromatic Imaging Camera)**:
    ```bash
    python fetch_epic_nasa_images.py
    ```

### Публикация в канал

*   **Автоматическая очередь**:
    ```bash
    python publish_all_images.py
    ```
    *Скрипт по кругу публикует все фото из папки `images` с интервалом, указанным в `.env`.*

*   **Публикация одного фото**:
    ```bash
    python publish_photo.py --image_path images/spacex_1.jpg
    ```
    *Если запустить без `--image_path`, скрипт выберет и отправит **случайное** фото из папки.*

> [!TIP]
> Используйте флаг `--help` у любого скрипта, чтобы увидеть полный список доступных настроек.
